# 任务交接：青少年科普竞技节目方案撰写（给 Codex）

## 一、背景

我们在为一档面向中小学生的“科普竞技闯关节目”设计单期方案。每一期围绕一个**孩子能亲手制作的小型实物作品**展开，作品要满足：

1. **真实小尺寸、当场能用**（不是大工程的缩小模型）；
2. **零 / 低编程门槛**——编程、接线、模块驱动由出题方用脚手架备好，孩子用免焊接、即插即用模块 + 可调参数完成；
3. **核心是开放式设计决策与思考**（阈值定多少、用声还是震、如何权衡），不是照说明书拼装；
4. 有**家国情怀 / 大国制造 / 民生关怀**的立意。

## 二、单期文档的固定结构（务必照此格式，参考“理学幻阵”模板）

1. 第一期内容（标题）
2. **基础材料**：富有文化 / 历史 / 科学色彩的科普长文（学习材料，选手学习时可用 AI 提问）
3. 一阶段答题 → 胜者从“资源包 A / 资源包 B”二选一（A = 额外硬件；B = 额外 3 分钟率先勘察场地，不计时、不可触碰）
4. **二阶段学习材料**：更深入的技术原理 + 该作品要权衡的几个难题
5. **项目小考**：一道有标准思考方向、但需推理的题（两队抢答），胜者得一阶段剩余资源包
6. **领队权益**：项目正式赛前，领队可对对方场地做一处干扰设置
7. **项目规则**（含项目名）+ 每队**套件清单表格**（模块 / 数量 / 说明）
8. **项目玩法细节详解**（设计组装 + 实地挑战，约 1 小时）
9. **现场流程**（逐条）+ 三级胜负判定（分数 → 次要指标 → 再次要指标）
10. 全程配 **4 张示意图**：原理图、关键难点图、套件散件图、现场布置 / 赛道图

## 三、产出要求

- 输出为 **Word（.docx）**，中文，正文宋体；标题用 Heading 1，小节标题加粗，条目用项目符号；图居中、配图注。
- 配图用代码生成示意图即可（中文字体可用 WenQuanYi Zen Hei / wqy-zenhei.ttc），逐张核对无重叠、无乱码。
- 套件清单用三列表格。
